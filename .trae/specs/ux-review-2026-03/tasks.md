# MDLooker 用户体验优化任务清单

## 第一阶段：核心功能（2 周）⭐⭐⭐⭐⭐

### Task 1: 市场准入一键查
**目标**：用户选择产品 + 国家后，显示完整的市场准入要求

- [ ] **Task 1.1**: 数据库设计
  - [ ] 创建 `market_access_guides` 表
  - [ ] 创建 `product_categories` 表（产品分类树）
  - [ ] 创建 `country_requirements` 表（各国要求）
  
- [ ] **Task 1.2**: 数据录入
  - [ ] 录入新加坡口罩准入要求（示例数据）
  - [ ] 录入马来西亚、泰国准入要求（对比用）
  - [ ] 录入 FDA、NMPA、HSA 等监管机构信息
  
- [ ] **Task 1.3**: API 开发
  - [ ] `GET /api/market-access?product=xxx&country=xxx`
  - [ ] `GET /api/product-categories`（产品分类树）
  - [ ] `GET /api/countries`（支持搜索）
  
- [ ] **Task 1.4**: 前端页面
  - [ ] 创建 `/market-access-wizard` 页面
  - [ ] 产品选择器组件（支持搜索/分类导航）
  - [ ] 国家选择器组件（支持多选）
  - [ ] 结果显示页面（完整报告）
  
- [ ] **Task 1.5**: 报告显示组件
  - [ ] 产品分类显示（带说明）
  - [ ] 资料清单（可勾选 + 下载模板）
  - [ ] 注册流程图（时间轴展示）
  - [ ] 费用明细表（官方费用 + 服务费）
  - [ ] 周期预估（最短/最长/平均）
  - [ ] 注意事项（警告提示）

**验收标准**：
- 用户能在 3 步内完成查询
- 显示完整的市场准入信息
- 至少包含 3 个国家的数据
- 页面加载时间 < 2 秒
