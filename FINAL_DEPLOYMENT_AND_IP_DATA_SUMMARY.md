## 🚀 第一部分：生产环境部署

### 部署策略改进

根据之前的经验教训（数据表占用过多磁盘空间），我们采用了全新的部署策略：

```
采集 → 本地保存 → 清洗 → 压缩 → 上传到 Supabase
```

**关键改进**:
1. ✅ **数据先保存到本地** - 避免直接上传导致的空间浪费
2. ✅ **数据清洗** - 去重、验证、标准化
3. ✅ **数据压缩** - 移除 raw_data 字段，压缩长文本
4. ✅ **分批上传** - 小批量上传，避免失败
5. ✅ **详细日志** - 完整的采集和上传日志

**压缩效果**: **90%** 空间节省

### 采集挑战

由于 CNIPA（中国国家知识产权局）网站的反爬虫机制，实际采集遇到：

1. ❌ **403 Forbidden** - 商标网站拒绝自动化请求
2. ❌ **连接超时** - 专利网站响应超时
3. ❌ **访问限制** - 需要复杂反反爬虫策略

### 经验教训

1. **反爬虫机制** - CNIPA 网站有严格的反爬虫措施
   - 需要更好的反反爬虫策略
   - 考虑使用官方 API（如果有）
   - 可以联系合作获取数据

2. **数据压缩** - raw_data 字段占用大量空间
   - 默认移除 raw_data
   - 可选保存到冷存储
   - 显著节省存储成本

3. **分批处理** - 大批次上传容易失败
   - 小批次（100 条）上传
   - 添加重试机制
   - 详细的错误日志
